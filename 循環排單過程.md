# 全局規則:
PLC在pOrderStatus.eProcessStatus = Init或Error時，PC應該取消Execute，並結束循環排單模式

# 流程
1. PC下pOrderControl.xOrderExecute時，PLC會返回pOrderStatus.eProcessStatus 在200~220的數據(定義參照enumOrderProcess_Status)，此時當作xBusy訊號
2. PLC返回pOrderStatus.eProcessStatus = WaitForCloseOrderExecute時，當作Done訊號，此時PC應該關閉Execute
3. PLC返回pOrderStatus.eProcessStatus = WaitForDownKnifeAndCrimper時，PC應該發送pOrderControl.xDownKnifeAndCrimper訊號，讓PLC下刀線
4. PLC將返回pOrderStatus.eProcessStatus = OrderInExecution ，表示訂單運作中
---
## 分支1
PC可根據需要發送pOrderControl.xChangeOrder，並同時發送新訂單數據
PLC在執行完換單完成訊號後會自動到pOrderStatus.eProcessStatus = OrderRecived狀態，繼續下一筆排單工作，並回到**流程2**

## 分支2
PC由人員操作，手動發送新一筆訂單(pOrderControl.xOrderExecute)，此時PLC會自動抬刀，回到**流程1**
