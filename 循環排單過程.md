# 全局規則:
PLC在pOrderStatus.eProcessStatus(1231) = Init(0)或Error時，PC應該取消Execute，並結束循環排單模式
僅當PLC返回pOrderStatus.xAllowTriggerOrderExecute(1230.3)時，PC發送pOrderControl.xOrderExecute(2070.0)才有作用

# 流程
1. PC下pOrderControl.xOrderExecute(2070.0)時，PLC會返回pOrderStatus.eProcessStatus(1231) 在200~220的數據(定義參照enumOrderProcess_Status)，此時當作xBusy訊號
2. PLC返回pOrderStatus.eProcessStatus(1231) = WaitForCloseOrderExecute(203)時，當作Done訊號，此時PC應該關閉Execute
3. PLC返回pOrderStatus.eProcessStatus(1231) = WaitForDownKnifeAndCrimper(210)時，PC應該發送pOrderControl.xDownKnifeAndCrimper(2070.1)訊號，讓PLC下刀線
4. PLC將返回pOrderStatus.eProcessStatus(1231) = OrderInExecution(212) ，表示訂單運作中
---
## 分支1
PC可根據需要發送pOrderControl.xChangeOrder(2070.4)，並同時發送新訂單數據，且須觸發pOrderControl.xHasNextOrder(2070.7)
PLC在執行完換單完成訊號後會自動到pOrderStatus.eProcessStatus(1231) = OrderRecived(200)狀態，繼續下一筆排單工作，並回到**流程2**

## 分支2
PC由人員操作，手動發送新一筆訂單(pOrderControl.xOrderExecute(2070.0))，此時PLC會自動抬刀，回到**流程1**
